Package Publisher

Generates a ZIP file package containing the specified files.

This publisher also allows the generation of a "manifest" to include in the package. A manifest contains additional details on the package, both at a general level and at a file level.

Version

This publisher is available from version 1.4.4.

Examples

Minimalist Example:

<package>
  <name>Example</name>
  <files>
    <file>Results.txt</file>
  </files>
</package>

Full example:

<package>
  <name>Example</name>
  <compression>9</compression>
  <always>true</always>
  <flatten>true</flatten>
  <single>true</single>
  <baseDirectory>C:\Builds\CC.Net</baseDirectory>
  <manifest type="defaultManifestGenerator" />
  <files>
    <file>Results.txt</file>
  </files>
</package>

Configuration Elements:

Node Description Type Required Default Version
name The name of the package file. If the extension .zip is omitted, it will be automatically appended. string Yes n/a  
compression The level of compression to use. The valid range is zero to nine, with nine being the most compression. int No 5  
always Always generate the package. By default the package will only be generated on a successful build. bool No false  
flatten Whether to include the file structure or not. By default the file structure will be included (based from base directory). If the structure is not needed, then setting this property will remove it. bool No false  
single This publisher also generates a list of packages generated. By default a list is generated in the artefacts folder, plus in an archive folder (one folder per build). Setting this property will only generate the main list and not generate per-build lists. bool No false  
baseDirectory The base directory to retrieve the files from. By default this will be the working directory. string No Working dir  
manifest The manifest generator to use. See Manifest Generators No None  
files The files to include in the package. These can be either absolute or relative paths - if relative it will be relative to the base directory Array of string Yes n/a  
dynamicValues Any dynamic parameters to be used. See Dynamic Parameters Dynamic values array - see Dynamic Parameters No None 1.5.0
