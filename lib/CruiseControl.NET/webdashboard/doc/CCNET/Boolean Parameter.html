Boolean Parameter

This will prompt the user to enter a boolean value when a force build is requested.

This parameter can then be used by a dynamic value in a task.

Version

This parameter is available from version 1.5.0.

Examples

Minimal example:

<booleanParameter>
  <name>Target</name>
  <true>Yes</true>
  <false>No</false>
</booleanParameter>

Full example:

<booleanParameter>
  <name>Target</name>
  <true name="PROD">Yes</true>
  <false name="DEV">No</false>
  <display>Production Build</display>
  <description>Do you want to generate a production build?</description>
  <default>DEV</default>
  <required>false</required>
</booleanParameter>

Configuration Elements:

Node Description Type Required Default Version
name The name of the parameter. This is the name that will be used internally by CruiseControl.NET. String Yes n/a 1.5.0
display The display label for the parameter. If set, this is what the user will see. If it is not set, then the name will be used for the label. String No none 1.5.0
description A description to display to the user. String No none 1.5.0
default The default value to use if the user does not enter a value for this parameter. String No None 1.5.0
required Whether the parameter is required or not. Boolean No False 1.5.0
true The value to use when the parameter is true. If the name attribute is set, then the user will see that as the selection value. Otherwise the actual value will be displayed. NameValuePair Yes n/a 1.5.0
false The value to use when the parameter is false. If the name attribute is set, then the user will see that as the selection value. Otherwise the actual value will be displayed. NameValuePair Yes n/a 1.5.0
