CruiseControl.NET : Modification Reader Task
This page last changed on Jul 20, 2009 by williams.

This tasks makes it possible to read back modifications made by the Modification Writer Task.
Available from 1.4 onwards.

Example :

2 projects in CCNet
1) is a project that does the compile, test, ... stuff, and also writes the modifications using the ModificationWriterTask
     be sure to set the appendTimeStamp of the modificationWriter to true
2) is a project that deploys the result of project 1
   --> copies it to other servers, updates source control (binary references like a framework), ...
   The reason for a second project is that this can be done on releases of milestones of project 1
   The ModificationReaderTask can now easily read the modification file(s) made by project one, into it's own integration,
   making it possible that these can be used by the existing tasks/publishers of ccnet for project 2

It is best to place the modificationreader in the prebuild section, so all the other tasks / publisers know the read modifications also.

Minimal configuration example:

<modificationReader />

Full configuration example:

<modificationReader>
     <filename>mods.xml</filename>
     <path></path>
</modificationReader>

Configuration Elements:

Node Description Type Required Default Version
filename The filename pattern for the file containing the modifications.
CCnet with search in the path for files starting with this filename, and having the same extention.
for example when filename is set to modifications.xml, ccnet will search for files like so :
modifications*.xml 		string false modifications.xml  
path The directory to search the xml file(s) in. string false the artifact directory  
deleteAfterRead Delete the files after they have been read.
bool false
false 1.4.4
description If filled in, this will be shown in the buildstage as the process name
string
false
n/a  
dynamicValues Any dynamic parameters to be used. See Dynamic Parameters Dynamic values array - see Dynamic Parameters No None 1.5.0

It is adivisable to keep these configuration elements of the modificationWriter and the modificationReader the same.
